# Agent Harness Core Invariants

Date: 2026-06-28

This catalog is the release-gate source for deterministic simulation, scenario replay, and review evidence. The same invariant IDs are exposed by `agent-harness invariants` through `agent_harness_core::quality::invariant_catalog`.

| ID | Invariant | Owner | Current staging evidence | Remaining live/soak evidence |
|---|---|---|---|---|
| I1 | One allowed inbound message triggers at most one model turn. | `channel_ingress`, `channel_runtime`, `runtime_queue` | Channel receive/enqueue tests; runtime queue idempotency tests. | Burst replay fixture with duplicate inbound events. |
| I2 | Every completed turn has exactly one delivery, explicit error notification, or dead-letter notification. | `runtime_pipeline`, `channel_delivery` | Runtime failure outbox tests; dead-letter timeout policy test. | Live provider timeout/dead-letter user notification sample. |
| I3 | Terminal states are irreversible. | `runtime_pipeline`, `runtime_queue`, `workers` | Timeout receipts are terminal for status/selection; queue retry creates a fresh id instead of resurrecting terminal state. | Operator retry/skip drill against staging queue. |
| I4 | Cancel/stop only affects the requested turn, queue item, worker job, or declared scope. | `admission`, `channel_state`, `runtime_queue`, `workers` | Scoped stop marker/receipt test; existing `/stop` cancel marker tests. | Live active-turn and active-job scoped stop receipt. |
| I5 | Crash recovery loses no work and duplicates no side effects. | `queue_shadow`, `supervision`, `runtime_worker` | Lease/idempotency tests; queue shadow divergence tests; supervisor breaker tests. | Seeded crash simulation and seven-day shadow parity summary. |
| I6 | Over-budget work is deferred or blocked, not dropped silently. | `autonomy`, `workers`, `mcp` | SQLite budget counter test; MCP allow-list rejection receipt. | Concurrent budget race fixture under load. |
| I7 | Every accepted ingress has a terminal trace chain reconstructable by one command. | `trace`, `status`, `logging` | `trace_harness_event` scans runtime/channel/log receipts and detects terminal status. | Live traceId sample across ingress, runtime, outbox, and delivery. |
| I8 | Agent identity and `/new` task boundaries are routing boundaries across channel state, session freshness, prompt, runtime, outbox, delivery, and memory namespaces. | `channel_state`, `runtime_pipeline`, `prompt`, `memory` | Non-main prompt isolation tests; `runtime_pipeline::tests::channel_session_freshness_does_not_cross_suppress_other_agent`; `prompt::tests::prompt_bundle_new_command_boundary_skips_prior_task_memory_context`; `memory::tests::non_main_memory_prompt_context_excludes_global_imported_snapshot_by_default`. | Live non-main same-DM replay with final outbox and delivery receipt; candidate-home smoke showing `/new` fresh prompt/memory task-boundary telemetry. |
| I9 | Final channel replies exclude progress and narration stream content when `assistantNarrationMode=progress_panel` is active. | `runtime_pipeline`, `progress`, `channel_delivery` | Defined by `docs/agent-harness-assistant-narration-tech-note.md` and `docs/agent-harness-channel-self-check.md`; Round10 adds `runtime_pipeline::tests::already_recorded_completion_repair_keeps_progress_panel_out_of_final_outbox` for the completion-repair path. | Full replay scenario with long progress narration, compact recovery fallback or fresh-thread recovery, short final answer, and Telegram/Discord outbox assertions. |
| I10 | Active Codex tool-use idle timeouts are stopped and routed through bounded recovery or an explicit terminal trace instead of directly dead-lettering the parent task. | `codex_runtime`, `runtime_pipeline`, `trace` | Round10 adds `codex_runtime::tests::run_codex_runtime_recovers_tool_use_idle_timeout_with_fresh_thread`, proving a stalled `commandExecution` item is stopped, recorded as `toolUseTimeout`, and continued through one fresh-thread recovery. | Broader per-tool cancellation/kill semantics, resumable tool supervisor support, and live long-review soak evidence. |
| I11 | Binary and bulky artifacts enter durable main-session context only as harness artifact references plus bounded extraction summaries; raw bytes, base64, provider URLs, and large tool blobs stay in artifact storage or receipts. | `media`, `prompt`, `runtime_worker`, `codex_runtime`, `workers`, `memory` | Round10 adds `media::tests::artifact_prompt_hygiene_renders_generic_bounded_summaries_not_payloads` and extends `runtime_worker::tests::prepare_runtime_queue_item_round_trips_inbound_media_artifacts_to_prompt`, proving image, audio/transcript, generated speech/media, browser capture, document/download, long tool-log/review transcript, worker-report, and provider-native media artifacts render as refs plus summaries without base64/provider payload leakage. | End-to-end artifact-heavy Telegram/Discord replay proving extraction writes bounded facts, raw artifacts remain retrievable by pointer, compact/fresh-thread recovery avoids reusing polluted context, and memory/continuity do not persist raw artifact blobs. |

Release rule: a regression that violates I1-I11 blocks cutover even if feature tests pass. I8 now includes `/new` as a prompt/memory task boundary, so fresh-task requests must not inherit previous-task imported memory context unless a future policy explicitly opts in. I9 now has a focused completion-repair regression gate; changes that touch compact fallback, fresh-thread recovery, or provider delivery still need the broader final-surface replay matrix before promotion. I10 is an initial guard, not full tool-supervisor parity; future work should add per-tool cancellation and resumable execution before claiming the gap closed. I11 is deliberately generic: image-only compact or image-only prompt redaction is not enough when audio, browser, document, worker, provider-native media, or large tool-log artifacts can enter durable context. Any new queue, tool, memory, or background-work path must either reuse these invariants or add a new invariant before release.
