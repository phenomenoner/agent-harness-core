Release rule: a regression that violates I1-I16 blocks cutover even if feature tests pass. I8 now includes `/new` as a prompt/memory task boundary, so fresh-task requests must not inherit previous-task imported memory context unless a future policy explicitly opts in. I9 now has focused completion-repair, stdout-recovery, structured worker-result redaction, read-only review suppression, and owner/session final-outbox routing gates; changes that touch compact fallback, fresh-thread recovery, provider delivery, or image/media tool completion still need the broader final-surface replay matrix before promotion. I10 is an initial guard, not full tool-supervisor parity; future work should add per-tool cancellation and resumable execution before claiming the gap closed. I11 is deliberately generic: image-only compact or image-only prompt redaction is not enough when audio, browser, document, worker, provider-native media, generated media previews, Discord/TG attachments, referenced media, or large tool-log artifacts can enter durable context. I12 is the bridge ownership guard: configured openclaw-mem write/recall must be primary, recall fallback must stay read-only, and store fallback must fail closed. I13 is the context-rollover guard: official compact success must either advance rollover accounting or leave a structured skip receipt before long-context continuity can be considered reliable; polluted inline/native-image contexts must rotate before another doomed turn; polluted-thread or stream-unstable recovery must not double-deliver, strand sibling pending items, resurrect a parent after child enqueue, or suppress final max-depth failure; `maxContinuationDepth` terminal cases must record `terminal-failed`; and model-visible continuity must stay bounded to resolver pointers rather than payload replay. I14 is the presentation guard: rich Telegram/Discord output must be adapter-rendered, capability-gated, safely escaped, receipt-accounted with render mode/fallback/full-text preservation, preserve attachment-index/artifactRef media refs, and traceable as a batch before it can replace plain text final delivery. I15 is the channel-session-history guard: "previous session", virtual-session working context, and cross-session search/retrieval paths must not return concrete wrong-lane history from blended agent space; only exact-lane resolver evidence or explicit project/global scopes may cross continuity boundaries, and broad scopes are bounded by the harness-home store partition. I16 is the outbound media guard: parser expansion, artifactRef resolution, and provider-native upload changes must ship with the same policy gate, protected-span masking, loud rejection, and policy/lint receipts. Provider request acceleration changes must prove `/fast` command replies stay model-turn-free, Codex app-server `serviceTier` is gated by model-catalog `serviceTiers`, supported routes send only camelCase `serviceTier`, and unsupported/proxy/native routes receive no unverified acceleration fields. Codex/Node service executable and startup changes must prove spawn-stable native resolution and terminal startup/write failure receipts before cutover. Any new queue, tool, memory, or background-work path must either reuse these invariants or add a new invariant before release.
