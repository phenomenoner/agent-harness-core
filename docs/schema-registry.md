Release rule: every new receipt/state schema must be added here and to `schema_registry_entries` before release. Breaking changes require a V2 schema, old-version reader, and migration receipt for one release cycle. A v0.8.0 reader accepting a legacy artifact is a forward-migration promise only; it does not mean a v0.7 binary can safely read state already written by v0.8.0.
