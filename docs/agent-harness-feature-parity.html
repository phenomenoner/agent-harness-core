Summary

Agent Harness 已可做受控 live Telegram/Discord handoff 測試，也能透過 Codex backend 跑基本 agent turns。這次升級補上 OpenRouter provider path，之後可讓像 小小梨 這類 agent 使用 OpenRouter 模型，同時保有各自的設定、session transcript、memory/canvas 空間，以及 provider-specific Codex home；先前 deferred 的 channel identity、delivery intent、runtime backoff、cron scheduler 項目也已進入 implemented scope。

OpenRouter route 由 harness 產生 provider-specific Codex config：codex-home-providers/openrouter、model_provider = "openrouter"、base_url = "https://openrouter.ai/api/v1"、env_key = "OPENROUTER_API_KEY"、wire_api = "responses"。密鑰不會寫入 config；preflight 會檢查 process env 或 harness secret env files，runtime/probe 會把必要 provider secret 注入 app-server child environment。

2026-06-12 hotfix triage 確認，新 agent/OpenRouter 設定曾把 model_provider = "openrouter" 寫入 shared Codex home，使 Telegram/Discord 的 default main turns 被錯路由到 OpenRouter 並因缺少 OPENROUTER_API_KEY 失敗。修復後，default Codex/OAuth route 會清掉 stale generated OpenRouter override；app-server method=error 或 turn/completed status=failed 會成為 ProtocolError receipt，不再產生 (no assistant text captured...) placeholder 回覆。Round4 之後，已知 app-server stream disconnect 會先 retry，再於耗盡嘗試後 dead-letter；其他 protocol/config failures 仍 terminal。

2026-06-12 ledger hygiene 將 JSONL append 集中到 serialized sidecar-lock writer，並讓 jsonl-repair 可拆回 }{ 這類同一行黏住的多筆 JSON value。Live repair 後 execution-receipts.jsonl、run-once-receipts.jsonl、harness.jsonl 均為 invalidLines=0，且原本黏住的紀錄被 recovered 而不是 dropped。

Response UX 現在分成兩層：assistantNarrationMode=progress_panel 只處理 Codex intermediate narration，emojiAccentMode=off 是預設；subtle 只在 explicit opt-in 且 successful final agent-reply outbox 寫入時加一個 guarded accent。Command replies、/status、error/failure replies、progress/status panels、code-heavy/fenced-code、risk/security/status-style replies、以及已經以 emoji 結尾的文字都不會被後處理污染；agent 與 channel 可以分別 override。Final Telegram/Discord replies 不再機械性加上 ◆ Agent header。

Channel identity 現在是 allow-list 之後的第二道 fail-closed gate：Telegram/Discord ingress 必須解析 platform/account/channel/thread binding，且 account selector 會一路保留到 queue/outbox/delivery。Provider reply target 由 harness 從 inbound provider ids 建立 deliveryIntent，不信任模型自己寫出的 quote/reply metadata。

Cron scheduler 現在由 cron-scheduler-lint、cron-scheduler-run-once 與 cron-scheduler-loop 負責 preflight 與 repeated due check：native 與 deterministic cron 來源會寫入 state/cron-scheduler 的 durable watermarks/receipts，並 idempotently enqueue WorkerStore jobs；worker-loop 仍然是唯一執行者。

Multi-agent isolation 現在涵蓋 registry/model override、runtime transcript 路徑，以及 prompt/lifecycle/canvas memory artifacts。檔案路徑會正規化 agent id，但 receipt 仍保留原始 Unicode 名稱，例如 小小梨。剩餘缺口標為 multi-agent-full-matrix-gap 與 per-agent-memory-recall-compartment-gap：每個 configured agent 仍需一個可重跑矩陣，且 public/non-main agents 需要 recall-policy proof，避免 global imported/private context 在未明確允許時進入 prompt context。

Virtual session continuity 已由 context rollover 實作：compact counters、virtual-session records、bounded working-set memory、continuation session index、prompt working-set injection、rollover episodes、guarded requeue receipts 都存在。2026-06-30 已在 live Discord/main lane 記錄到首次真實 rollover：門檻達到（successfulCompactCount=2）、applied receipt 把 cont-1 re-key 成 cont-2（同一 virtual session），並寫出 virtual-session、episode-index 與 continuation-rollover working-set receipts。剩餘的 virtual-session-continuity-gap 收斂為：一個 end-to-end scenario 證明 continuation turn 的 working-set prompt injection 一路到 final delivery，並把最終回覆 trace 回 continuation queue item。

OpenClawMem 現在以 harness-side service adapter 完成：CLI 與 sidecar 均支援 status、recall、reviewed propose、approved store、per-agent writeback 與 receipts。實際匯入內容確認是 Qdrant edge preserved snapshot、SQLite、JSONL 與 engine state；若 openclaw-mem-engine bridge/sidecar 缺席，正確回報是 explicit read-only migration-fallback，不是 full graph autonomous matching。新 source builds 可透過 memory-owner-local-prepare 加 operator-approved promotion 走本機 in-process owner 路徑，promotion 後報告 local-in-process；完整 parity 仍需 bridge envelope、native/Qdrant proof、routeAuto/autonomous matching fixtures、provenance/freshness receipts 全綠。

未能以 code-only 完成的項目已移到 live/soak gates：real plugin invocation、Windows service-wrapper registration/reboot proof、repo-local encrypted vault live migration，以及需要 operator 發送的 Telegram/Discord/OpenRouter live smoke。Native non-OpenAI provider adapters 已依 2026-06-12 使用者決策移到 Deferred backlog，短期不阻塞 OpenRouter-centered provider path。若未來要升級到 remote openclaw-mem service，需補 endpoint、API/wire contract、credentials 與 fixtures。

2026-06-12 外部 roadmap 已整合進 docs/agent-harness-core-roadmap-backlog.md：P0-P7、Track T、Track M、Icebox 全部都有實作步驟、驗收標準、staging implementation status 與 pending live gates。staging pass 已補上 fail-closed config、retry/dead-letter、log rotation、queue retry/skip、supervision evaluation、health/trace/metrics、canary receipts、queue shadow、background registry、token gates、learning proposals、MCP request handling、repo-local encrypted vault、security scans、invariants、schema registry、release docs 與 operator CLI gates。

驗收標準已對齊外部 comparative review 的十個面向：concurrency、persistence、error recovery、supervision、security、observability、token efficiency、extensibility、testing quality、maturity。Deterministic code paths 以 tests/fixtures 為證據；queue shadow parity、live channel smoke、reboot recovery、canary deploy、advisory audit、30-day SLO 這類項目必須等 live/soak receipts，不能只靠本機測試宣稱完成。

Round4 reconnect/tone 驗證已通過 cargo fmt、cargo test -p agent-harness-core（core 219）、cargo test -p agent-harness-cli（CLI 17）、staging/canonical cargo build -p agent-harness-cli、git diff --check、tracked-file public export public-hygiene（forbiddenHits=[]）、live config-validate、live harness-skills-sync、live healthz 與 live status --json。Channel identity / delivery intent / cron scheduler pass 已通過 cargo fmt、cargo check、cargo test（core 229、CLI 18、doctests 0）、git diff --check、以及 tracked-file public-hygiene（forbiddenHits=[]）。Round4-2 live-control/native cron pass 已通過 cargo fmt --all --check、staged workspace check、core 239、CLI 18、staged build、public export hygiene，以及 non-live cutover CLI smoke。Round4-3 robustness pass 補上 runtime-loop safe-mode/no-silent-failure、cron scheduler lint/interval hardening、bounded status/readiness scans；最終驗證記錄在 operations handbook。

Roadmap Notes

  • Accepted correction: round3-2 B1 background lifecycle state goes into SQLite WorkerStore; JSONL remains audit/receipt only.
  • Native MCP is adopted, but the first pass should target initialize/list/call fixtures rather than every optional MCP feature.
  • Warm app-server pooling is gated by measurement and must prove no session, cwd, auth, or agent-context contamination.
  • Canary deploy must support fake-only canary when live provider credentials are unavailable.
  • /healthz must be local/admin-gated by default and never expose secrets or raw prompt/memory content.

Deferred / Icebox

  • Native provider adapters beyond Codex/OpenAI/OpenRouter-compatible routing: user confirmed on 2026-06-12 that this is needed, but should be done later. When resumed, provide provider list, SDK/API contract, credentials, model naming rules, cancellation/streaming expectations, and test fixtures.
  • Icebox: full Tokio/async rewrite, new channels, TypeScript plugin SDK rewrite, web dashboard, multi-user/multi-tenant mode, cloud deployment, in-house vector/rerank/memory governance, and benchmark leaderboard chasing.

Foundation First

  1. Close atomic write, retry/dead-letter, config, queue operator commands, and log-bound gaps.
  2. Start invariants and deterministic simulation before the queue migration.

Supervise / Trace

  1. Implement supervise, restart policy, heartbeat, alerts, ordered shutdown, and OS persistence.

P2

  1. Dual-write JSONL and SQLite channel turns.
  2. Require seven days zero divergence before SQLite becomes execution source.

P3

Jobs / Semantics

  1. Add scoped stop, collect/interrupt, per-session serial lanes, rate/admission control, and long-task contract.

P4

Token Efficiency

  1. Aggregate token telemetry and prove stable-context dedup.
  2. Add recall ledger, progressive skill disclosure, job-kind routing, session rotation, and golden prompt bundles.
P5 + M/P6/P7

Autonomy With Gates

  1. Add tasks, heartbeat, budgets, native MCP request handling, future stdio serve, chaining, drift, and learning-loop proposals with learning default-on in propose-only mode.
  2. Run memory contracts, security hardening, CI, schema registry, release, and public hygiene tracks in parallel where gated.
