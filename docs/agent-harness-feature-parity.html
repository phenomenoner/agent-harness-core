Summary

Agent Harness 已可做受控 live Telegram/Discord handoff 測試，也能透過 Codex backend 跑基本 agent turns。這次升級補上 OpenRouter provider path，之後可讓像 小小梨 這類 agent 使用 OpenRouter 模型，同時保有各自的設定、session transcript 與 memory/canvas 空間。

OpenRouter route 由 harness 產生 repo-local Codex config：model_provider = "openrouter"、base_url = "https://openrouter.ai/api/v1"、env_key = "OPENROUTER_API_KEY"、wire_api = "chat"。密鑰不會寫入 config；preflight 只檢查環境變數是否存在。

Multi-agent isolation 現在涵蓋 registry/model override、runtime transcript 路徑，以及 prompt/lifecycle/canvas memory artifacts。檔案路徑會正規化 agent id，但 receipt 仍保留原始 Unicode 名稱，例如 小小梨。

未能直接完成的項目已移到 Blocked/Action Required：native non-OpenAI provider adapters、Qdrant/openclaw-mem service parity、real plugin invocation、Windows unattended service registration、vaulted secrets migration，以及需要 operator 發送的 live smoke。

本輪本機驗證已通過 cargo fmt --all、cargo test --workspace（core 177、CLI 16、doctests 0）、cargo build --workspace、live status、live enable-check 與 channel outbox plan。

  • Plugin real invocation: approve plugin trust model, allow-list, timeout policy, per-agent/channel permissions, and fixtures.
