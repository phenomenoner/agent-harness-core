Executive Summary

目前 harness 已可進行 live Telegram/Discord handoff 測試。它能匯入主要 OpenClaw 狀態、保留多 agent registry、處理 channel commands、接 DM、組 prompt、用 Codex app-server 跑 turn、寫 transcript/trajectory/receipts/logs。

最大缺口不是外層 shell，而是更深的 OpenClaw 行為：Qdrant-native memory、完整 openclaw-mem routeAuto/propose/store/canvas、plugin hook/tool execution、cron execution、subagent execution，以及 durable Windows service supervision。

Current Warnings

  • Discord 還需要 allowed-user real inbound DM receipt。
  • Memory lifecycle 要等下一個成功 live agent turn 寫 receipt。
  • LanceDB 缺席，但 Qdrant edge 是 primary，所以目前是 optional。

Recommended Development Order

P0

Finish live handoff proof

  1. Telegram + Discord allowed-user normal turn smoke。
  2. 確認 memory lifecycle receipt 寫出。
  3. 清 Discord real inbound warning。
  4. 把 smoke transcript/receipts 寫回 activation plan。
P1

Make it durable on Windows

  1. 註冊 Task Scheduler 或改 service wrapper。
  2. restart policy、stale heartbeat monitor、單一 stop/start/status。
  3. log rotation 與 operator summary。
  4. backup/export + cutover receipt。
P2

Close memory parity

  1. Qdrant-native recall sidecar/service。
  2. LanceDB fallback。
  3. routeAuto/autoRecall + context budget。
  4. propose/store review gates。
  5. 完整 symbolic canvas / graph behavior。
P3

Close plugin parity

  1. Sidecar real tool invocation。
  2. prompt/tool-result/agent-end hooks。
  3. 優先 openclaw-mem / mem-engine / context-budget。
  4. Timeouts、permissions、execution receipts。
P4

Cron and subagents

  1. Native agent-turn cron scheduler。
  2. Deterministic cron runner with no LLM path。
  3. Subagent worker execution / resume / cancel。
  4. cron-to-agent 與 parent-to-subagent live smoke。
P5

Broaden provider parity

  1. OpenRouter-compatible live validation。
  2. Provider-specific adapters where Codex is insufficient。
  3. Per-agent credential routing。
  4. Model/thinking policy compatibility tests。

Current Risks

Discord inbound connected, but still needs real allowed-user DM receipt.
Memory recall is SQLite-vector first; Qdrant edge is not native-read yet.
Plugin support is catalog/bridge-first, not full hook/tool execution.
Scheduled Tasks are generated but not registered in this environment.
/stop needs stronger hard cancellation for already-running Codex turns.
Long-lived secrets should move to Windows Credential Manager or encrypted vault.
