The human analogy

Experience becomes useful only after reflection

People combine teaching, documentation, demonstrations, and other people's lessons with their own attempts. We retain the procedure that worked, the conditions that mattered, and the warning signs that prevented failure. A skill gives an agent the same kind of reusable procedural asset—without confusing one episode with a universal rule.

The goal is not an agent that rewrites itself after every turn. It is an evidence-governed system that learns the smallest reusable procedure that improves the next comparable task.

Expected benefit

  • better task outcomes and verifier results
  • fewer corrections and repeated failures
  • faster, more consistent execution
  • capability retained across future tasks
−

Total cost and risk

  • context, retrieval, latency, and tool cost
  • irrelevant-procedure interference
  • evaluation and maintenance burden
  • safety, authority, and external-effect risk

Role-first architecture

Find less. Help more.

The system does not start by scanning an entire library into the prompt. It asks what competence role is unresolved, applies hard identity and capability boundaries, and discloses only the fragment whose expected marginal value is positive.

1

Bound the task

Objective, stage, constraints, risk, tools, authority, and available evidence.

2

Identify roles

Diagnosis, planning, domain execution, verification, safety, or delivery.

3

Filter hard

Exact agent, account, channel, user, tool, lifecycle, and permission eligibility.

4

Compare value

Revision-specific utility, cost, uncertainty, freshness, and relationships.

5

Disclose and stop

The smallest sufficient fragment—or no skill when none improves the task.

Residency policy

Availability is not the same as relevance

Competence describes what a procedure can do. Residency describes when it may become model-visible. The two axes are measured independently so frequency cannot quietly turn into permanent prompt residency.

Always scarce

Resident kernel

A tiny capsule loaded by default only when broad outcome lift exceeds its repeated context and interference cost.

Outside context

Priority / predictive

Rank metadata or prefetch content when expected latency savings exceed waste. Prefetch is not disclosure or use.

Normal path

Role on-demand

Load the smallest core or reference slice after the current task demonstrates an unresolved competence role.

Authority required

Explicit / privileged

Prediction cannot bypass an explicit invocation, permission, approval, or external-effect boundary.

Experience loop

Learn from outcomes, not activity

A skill being surfaced, read, followed, and causally useful are different facts. Agent Harness Core keeps them separate, joins them to effects and outcomes, and preserves uncertainty when several skills contributed.

Observe

Record candidate, disclosure, material use, test, effect, correction, cost, and terminal outcome.

Attribute

Localize evidence to retrieval, role, procedure, composition, capability, or no reusable change.

Reflect

Update contextual beliefs per task; periodically review repeated patterns, drift, conflicts, and residual gaps.

Deploy behavior

Propose, independently evaluate, shadow, promote narrowly, observe, retain, or roll back.

Contextual beliefs

No global “best skill” score

Hierarchical Bayesian updates can share cautious priors across related roles while retaining revision-, context-, backend-, and time-specific uncertainty.

Beyond software

A reusable capability asset for everyday work

External guidance and individual experience can reinforce each other in daily life, interpersonal work, and professional operations—as long as facts, private observations, policy, and reusable procedure remain properly separated.

Life

Trip preparation

Combine public guidance with observed packing mistakes, accessibility needs, timing, and verification results. Current bookings stay task data; only the reusable preparation flow becomes a skill.

Interpersonal

Difficult feedback

Use communication frameworks plus accepted revisions and corrections to improve a scoped preparation procedure—without inferring private psychology or universalizing one interaction.

Work

Incident response

Join vendor documentation, prior failure modes, recovery evidence, and post-incident review. Environment drift lowers confidence and triggers revalidation instead of replaying an obsolete runbook.

Decision science

Measure capability, not churn

Baselines, task slices, metrics, costs, observation windows, and uncertainty rules are frozen before results are inspected. A lower selection count, a cleaner skill body, or more proposals is not automatically an improvement.

Outcome

Acceptance, verifier pass, corrections, failures, time to terminal, and comparison with no-skill or prior-revision baselines.

Coverage & abstention

Required-role recall, critical-role coverage, no-skill accuracy, false disclosure, and confidence calibration.

Disclosure efficiency

Model-visible skill tokens, irrelevant fragments, repeat loads, and verified uplift per 1,000 skill tokens.

Total cost

Latency, tools, retries, evaluator spend, operator attention, maintenance churn, and opportunity cost.

Evolution quality

Held-out regression, false promotion, lost coverage, attribution overturn, drift, and rollback success.

Hard boundaries

Cross-identity exposure, unauthorized effects, policy bypass, secret leakage, and irreversible mutation must remain zero.

Organic growth means

  • versioned procedural assets
  • evidence-aware role and residency changes
  • several revisions or a new skill when evidence supports them
  • independent evaluation and exact rollback

Organic growth does not mean

  • rewriting policy from one successful task
  • equating popularity with quality
  • merging skills because they look similar
  • letting the task model approve its own permanent change

Delivery path

Earn autonomy one boundary at a time

Each expansion of authority is a behavioral deployment with its own evidence. The project moves from observation to narrow promotion only when the affected risk class is independently qualified.

Correct and observe

Remove unattributed mutation, compile exact contracts, and compare new policies with no serving-side effects.

Join and evaluate

Connect real effects and outcomes, calibrate beliefs and evaluators, and test proposals against frozen held-out cases.

Promote narrowly

Activate one reversible low-risk cohort, verify provider-visible outcomes, and widen only with evidence.

From a static agent to a system that can accumulate individual capability.

The model remains powerful but interchangeable. Agent Harness Core supplies the durable identity, evidence, decision, and rollback layer that lets experience improve future work without turning learning into an uncontrolled rewrite loop.
